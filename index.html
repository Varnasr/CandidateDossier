Why CSV, why blocks, and why masking?

We generate candidate dossiers by turning a structured CSV into a LaTeX blocks file (candidate_blocks.tex), which is then included in a simple LaTeX template to export a high-quality PDF. “Blocks” let you reuse the same layout across dozens of candidates without hand-editing.

Template looks like
main.tex includes \input{candidate_blocks.tex} inside a section.
Blocks contain
Per-candidate tables and paragraphs (rationale, strengths, proposers…).
CSV fields
preference,name,category,caste,age,occupation,education,joined,phone,record,rationale,strengths,weaknesses,proposers,positions,elections

Phone numbers are mandatorily masked (e.g., ******2345) to protect privacy and to comply with Indian data norms. Masking happens automatically during block generation.

Backup (Overleaf)

If network or serverless compilation is unavailable, use Overleaf:

  1. Download main_template.tex and generate a candidate_blocks.tex (Easy tab can export blocks only if you prefer).
  2. Go to Overleaf → New Project → Upload Project → add both files.
  3. Recompile and download your PDF.

Phone masking is already enforced in our block generator.
